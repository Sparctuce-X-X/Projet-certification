# Migrations Supabase — Index complet

> Source de vérité de l'historique DB. Référencé depuis `CLAUDE.md` § Migrations Supabase.
> Mis à jour à chaque nouvelle migration jouée en prod.

## 🔴 Règle d'or — base de données incrémentale

**On crée les tables au fur et à mesure des features, jamais en bloc.** Une feature qui n'est pas encore codée n'a pas sa table en DB. La DB Supabase ne contient à tout moment que ce qui est consommé par le code en place.

**Conséquence** : `docs/references/niqo_schema_v1.6.sql` est une **référence partielle** (tables `transactions` et `litiges` supprimées en v4.0). On en extrait par feature, dans une migration numérotée.

## Convention de fichiers

```
docs/migrations/
  01_users.sql → 109_cron_instrumentation.sql (jouées)
  110_<feature>.sql (à créer)
  ...
```

- Numérotation séquentielle `NN_feature.sql`, jamais réutilisée.
- Chaque migration est **idempotente**.
- À jouer manuellement dans Supabase Dashboard → SQL Editor.
